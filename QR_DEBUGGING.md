# QR Code Debugging Summary

## Problem

The QR code generated by the Monkey C implementation was not scannable. Visual inspection showed the QR code looked similar to a valid one but had subtle differences throughout the data region.

## Root Causes

Three issues were identified and fixed:

### 1. Missing Alignment Pattern

**Problem:** Version 2 QR codes require a 5×5 alignment pattern centered at position (18, 18). This pattern was:
- Not being drawn in the matrix
- Not being excluded during data placement

**Impact:** Data was being placed in 384 positions instead of the correct 359, causing all data bits to be shifted and corrupting the entire QR code.

**Fix:** Added `addAlignmentPattern()` function and updated `isFunctionModule()` to exclude the alignment pattern area during data placement.

```monkeyc
// Added alignment pattern check in isFunctionModule()
if (mVersion >= 2) {
    var alignCenter = getAlignmentPatternCenter();
    if (alignCenter > 0) {
        var dr = row - alignCenter;
        var dc = col - alignCenter;
        if (dr >= -2 && dr <= 2 && dc >= -2 && dc <= 2) {
            return true;
        }
    }
}
```

### 2. Wrong Mask Pattern

**Problem:** The code was using mask pattern 0, but the Python qrcode library (our reference) uses mask pattern 5 for the test string "A3163889".

**Impact:** All data modules were XORed with the wrong pattern, inverting bits in different positions than expected.

**Fix:** Changed `MASK_PATTERN` constant from 0 to 5 and implemented all 8 mask pattern formulas.

```monkeyc
private const MASK_PATTERN = 5;

private function shouldMask(row, col, pattern) {
    switch (pattern) {
        case 5: return ((row * col) % 2) + ((row * col) % 3) == 0;
        // ... other patterns
    }
}
```

### 3. Incorrect Format Information Placement

**Problem:** The format information bits were being placed in the wrong positions around the finder patterns.

**Impact:** Scanners couldn't decode the error correction level and mask pattern, making the QR code unreadable.

**Fix:** Corrected the bit placement positions according to the QR code specification:
- Horizontal placement in row 8, skipping column 6 (timing pattern)
- Vertical placement in column 8, skipping row 6 (timing pattern)
- Duplicate placement around bottom-left and top-right finders

## Data Capacity Verification

For Version 2, Level L:
- Total modules: 25×25 = 625
- Function pattern modules: 266 (finders + separators + timing + alignment + dark module)
- Data modules: 359
- Data capacity: 352 bits (44 codewords × 8) + 7 remainder bits
